## 27. 바이브코딩

- 바이브코딩
- ２０２５．０２．０３
- 가능해지는 일
- 코딩없이 제품개발을 가능하게 함
- 자연어로 제품을 만들 수 있음
- 개발 지식 없이도 동작하는 웹앱 생성.
- → 당신의 아이디어 빠르게 검증 가능!
- 그냥 느낌 가는 대로 코딩하는 거. 요즘 나는 이걸 “vibe coding”이라고 부르고 있어요. 코드가 실제로 어떻게 생겼는지는 거의 신경 안 씁니다.
- 변경사항? 안 봐요. 그냥 “Accept All”. 에러 나면? 에러 메시지 복붙해서 넣고 아무 말도 안 해요. 근데 대부분 알아서 고쳐줘요. 진짜.
- 이렇게 코드를 만들다 보면, 어느 순간 내가 뭘 짰는지 이해 못할 정도로 커 있어요. 사실 좀 무섭죠. 그래서 코드 리뷰는 커녕, 읽지도 않아요.
- 왜냐면 이제 LLM들이 너무 잘하거든요. Cursor Composer, Sonnet 이런 애들 덕분에 가능한 시대예요. 저 같은 경우는 그냥 Composer에다 SuperWhisper로 말만 해요. “사이드바 패딩을 반으로 줄여줘” 같은 진짜 시시한 것도요. 너무 귀찮아서 직접 찾기도 싫고.
- 가끔 모델이 못 고치는 버그 나오면? 그냥 적당히 우회하거나 “이거 저거 바꿔봐” 하면서 실험해요. 되면 다행이고, 안 되면 또 복붙하고.
- 물론 이게 진짜 프로덕션 코드 짜는 건 아니에요. 근데, 주말에 뚝딱 만드는 실험용 웹앱이나 프로젝트엔 딱이에요.
- 결론? 그냥 보고, 말하고, 복붙하고, 실행하고. 근데 놀랍게도… 꽤 잘 돌아갑니다.
- 안해보셨다면
- 많이 놓치시는겁니다.

**Speaker notes**
- 할루시네이션 이란 말 만들어낸사람

- User Persona 설정
- 경쟁 서비스 리서치
- 요구사항 목록 작성 (Requirement List)
- 기술 스택 후보 정리

- 🛠

- Frontend 구조 설계 (React, Vue, etc.)
- Backend API 설계 및 DB schema 작성
- 인증, 보안 설정
- CI/CD 설정 및 도메인 연결
- 초기 배포 (Netlify, Vercel, Render 등)
- 🚀
- 5. MVP + Market Testing
- MVP 기능만 우선 개발 (핵심 기능 집중)
- 베타 사용자 모집
- 온보딩 UX 테스트
- 시장 반응 수집 (인터뷰, 설문, 히트맵 등)
- 버그 트래킹 시스템 운영 (Jira, GitHub Issues)
- 🔁
- 6. Feedback Loop & Iteration
- 피드백 기반 기능 수정
- 버그 수정 및 성능 개선
- 사용자 유지 전략 적용 (리텐션 개선)
