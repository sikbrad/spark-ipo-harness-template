# SOLAPI 카카오톡 첨부파일 발송 가능 여부와 비용 리서치

작성일: 2026-06-08
질문: SOLAPI API로 이미 고객에게 카카오톡 메시지를 보내고 있는데, 첨부파일도 보낼 수 있는가? 가능하다면 비용은?

## 결론

가능한 것은 "일반 파일 첨부"가 아니라 "이미지형 메시지" 또는 "파일 다운로드 링크 버튼/URL 포함"이다.

- PDF, Excel, 견적서, 계약서 같은 일반 파일을 카카오톡 메시지에 실제 첨부파일처럼 붙여 보내는 SOLAPI 카카오 발송 필드는 공식 문서에서 확인되지 않는다.
- 알림톡은 이미지첨부형/이미지 관련 템플릿을 쓸 수 있고, 발송 API의 `kakaoOptions.imageId`를 사용할 수 있다. 단, 템플릿 사전 검수와 이미지 규격을 맞춰야 한다.
- 브랜드 메시지는 이미지형, 와이드, 캐러셀 등 리치 버블을 쓸 수 있고, 자유형 이미지 발송은 `kakaoOptions.bms.chatBubbleType: "IMAGE"` + `kakaoOptions.bms.imageId` 구조다.
- 일반 파일을 보내야 한다면 파일을 우리 서버/S3/OneDrive/Drive 등에 올린 뒤, 알림톡/브랜드 메시지 버튼 또는 본문 URL로 다운로드 링크를 보내는 방식이 현실적이다.
- 친구톡은 카카오 정책상 2025-12-31 종료되었고, 2026-01-01 이후 친구톡/이미지 친구톡 요청은 브랜드 메시지 자유형으로 자동 전환된다. 기존 코드가 친구톡을 보내고 있다면 현재 과금/동작은 브랜드 메시지 자유형 기준으로 봐야 한다.

## 가능 범위

### 1. 알림톡: 정보성 메시지 + 사전 승인 템플릿

알림톡은 주문, 예약, 결제, 배송, 안내 같은 정보성 메시지에 적합하다. 채널 친구 추가 없이 수신번호 기반 발송이 가능하지만, 사전 승인된 템플릿으로만 발송할 수 있다.

SOLAPI 문서 기준 알림톡 발송 형식에는 기본형, 강조표기형, 이미지첨부형, 리스트형이 있다. 파일 업로드 API에도 카카오 알림톡 이미지 타입이 따로 있다.

구현 포인트:

- 이미지 업로드: `POST /storage/v1/files`
- 업로드 타입:
  - `KAKAO`: 카카오 알림톡 이미지, 500KB, png/jpg, 최소 500x250px, 비율 2:1~1:1.5
  - `ATA`: 카카오 알림톡 첨부 이미지, 500KB, png/jpg, 최소 너비 500px, 비율 정확히 2:1
- 발송: 메시지 발송 API에서 `kakaoOptions.pfId`, `kakaoOptions.templateId`, 필요 시 `kakaoOptions.imageId` 사용
- 주의: 알림톡은 템플릿 기반이므로 이미지형으로 바꾸려면 템플릿 등록/검수가 필요하다.

비용:

- 표준 단가: 13원/건, VAT 별도
- SOLAPI 요금표는 알림톡에 대해 "유형 관계없이 1건당 13원 동일"이라고 안내한다.
- 월 발송량 할인이 적용되면 10,000~99,999건 10원, 100,000~499,999건 8원, 20,000,000건 이상 5.5원까지 내려간다.

판단:

- 고객에게 정보성 안내와 함께 대표 이미지 1장을 보여주는 용도면 알림톡 이미지형이 가장 저렴하다.
- PDF/Excel 자체 첨부에는 맞지 않는다. 이 경우 버튼으로 다운로드 링크를 연결해야 한다.

### 2. 브랜드 메시지: 채널 친구/수신동의 대상 리치 메시지

브랜드 메시지는 텍스트, 이미지, 와이드, 캐러셀 등 다양한 버블 타입을 지원한다. SOLAPI 카카오톡 API 문서상 브랜드 메시지는 알림톡과 달리 다양한 `chatBubbleType`을 지원한다.

브랜드 메시지 자유형 이미지 발송 구현 포인트:

```json
{
  "messages": [
    {
      "to": "01012345678",
      "text": "이미지형 브랜드 메시지 예시입니다.",
      "kakaoOptions": {
        "pfId": "PFxxxxxxxxxxxxxxxx",
        "bms": {
          "targeting": "I",
          "chatBubbleType": "IMAGE",
          "imageId": "IMGxxxxxxxxxxxxxxxx",
          "buttons": [
            {
              "name": "다운로드",
              "linkType": "WL",
              "linkMobile": "https://example.com/file"
            }
          ]
        }
      }
    }
  ]
}
```

이미지 업로드:

- `POST /storage/v1/files`
- `BMS`: 브랜드 메시지 이미지, 5MB, jpg/png, 권장 800x400px, 비율 2:1~3:4
- `BMS_WIDE`: 브랜드 메시지 와이드 이미지, 5MB, jpg/png, 권장 800x600px, 비율 2:1~1:1

타겟팅:

- `I`: 채널친구, 브랜드 등록 불필요
- `M`: 마케팅 수신 동의자, 브랜드 등록 필요
- `N`: 마케팅 수신 동의자 중 채널친구 제외 성격, 브랜드 등록 필요

비용:

- 브랜드 메시지 텍스트: 53원/건, VAT 별도
- 브랜드 메시지 이미지: 83원/건, VAT 별도
- 브랜드 메시지 와이드: 88원/건, VAT 별도
- 브랜드 메시지 자유형(텍스트/이미지): 100원/건, VAT 별도
- 브랜드 프리미엄: 103원/건, VAT 별도
- 2026-01-01 이후 친구톡 요청은 브랜드 메시지 자유형으로 전환되며, SOLAPI 공지 기준 자유형 텍스트/이미지 단가는 100원/건이다.
- 월 발송량 할인표 기준 브랜드 자유형은 10,000~99,999건 90원, 100,000~499,999건 85원, 20,000,000건 이상 69원까지 내려간다.
- 브랜드 이미지형은 10,000~99,999건 74원, 100,000~499,999건 69원, 20,000,000건 이상 59원까지 내려간다.

판단:

- 마케팅/프로모션/채널친구 대상 이미지 메시지에는 브랜드 메시지가 맞다.
- 기존 친구톡 코드가 있다면 지금은 브랜드 메시지 자유형 자동 전환 영향과 단가 100원을 먼저 확인해야 한다.
- 일반 파일은 여전히 실제 첨부가 아니라 버튼/이미지 클릭 링크로 보내는 편이 맞다.

### 3. 문자 MMS/RCS: 카카오톡이 아니어도 되는 경우

카카오톡이 꼭 아니고 "이미지 첨부 문자"가 목적이면 MMS 또는 RCS MMS가 가능하다.

SOLAPI 메시지 발송 문서 기준:

- MMS 필수 필드: `from`, `to`, `text`, `imageId`
- MMS 이미지: JPG, 최대 200KB
- 표준 MMS 단가: 110원/건, VAT 별도
- 월 발송량 할인표 기준 MMS는 10,000~99,999건 90원, 100,000~299,999건 80원, 4,000,000건 이상 55원까지 내려간다.

단, 이것은 카카오톡 발송이 아니라 문자 발송이다.

## 일반 파일 첨부가 안 된다고 보는 근거

SOLAPI 스토리지 API에는 `DOCUMENT` 타입이 있고 PDF/TIF 등 일반 문서를 업로드할 수 있다. 하지만 메시지 발송 API의 카카오 관련 필드에는 일반 문서 `fileId`를 붙이는 구조가 보이지 않는다.

공식 문서상 발송 쪽에서 확인되는 첨부/미디어 구조는 다음 정도다.

- MMS: `imageId`
- 알림톡: `kakaoOptions.imageId`
- 브랜드 메시지 이미지형: `kakaoOptions.bms.imageId`
- 팩스: `faxOptions.fileIds`

따라서 `DOCUMENT` 업로드는 증빙/관리/팩스 등 다른 용도일 수 있고, 카카오톡 메시지에 PDF/Excel을 직접 첨부하는 방식으로 해석하면 안 된다. 카카오톡 고객 발송에서 일반 문서를 보내려면 다운로드 링크 버튼/URL 방식으로 설계하는 것이 안전하다.

## 대체발송 비용 주의

알림톡, 브랜드 메시지 자유형 등은 실패 시 문자 대체발송 설정이 영향을 준다.

- 알림톡은 기본적으로 대체 발송이 활성화되어 있으며, `kakaoOptions.disableSms: true`가 아니면 실패 시 문자 대체를 시도한다.
- 대체발송이 정상 동작하려면 `from`에 사전 등록된 발신번호가 필요하다.
- 대체 발송 시 원래 타입 가격은 환불되고 문자 타입 비용이 차감된다.
- 브랜드 메시지 자유형 이미지가 LMS로 대체되면 이미지는 MMS가 아니라 URL 링크 형태로 LMS 본문에 포함된다.
- LMS 표준 단가: 45원/건, VAT 별도

## 추천 설계

### 정보성 알림 + 파일 링크

가장 추천한다.

- 알림톡 템플릿 본문에 핵심 안내
- 버튼명: `견적서 보기`, `계약서 확인`, `배송자료 확인` 등
- 버튼 URL: 우리 서버나 스토리지의 만료형 다운로드 링크
- 필요하면 이미지첨부형 템플릿으로 대표 이미지/썸네일 추가

비용은 보통 알림톡 13원/건(VAT 별도) 기준이다. 대량 발송이면 구간 할인 적용 가능성이 있다.

### 이미지 자체가 메시지의 핵심인 경우

- 정보성: 알림톡 이미지첨부형 템플릿
- 광고/마케팅/채널친구 대상: 브랜드 메시지 이미지형 또는 자유형 이미지

비용은 알림톡이면 13원/건, 브랜드 이미지/자유형이면 83원 또는 100원/건 기준으로 잡는다.

### PDF/Excel을 꼭 파일처럼 보내고 싶은 경우

SOLAPI 카카오 발송으로는 직접 첨부 대신 링크로 보내는 설계를 권장한다.

운영 주의:

- 공개 URL 대신 만료형/토큰형 링크 사용
- 고객별 파일이면 고객 인증 또는 난수 토큰 적용
- 개인정보 포함 파일이면 접근 로그와 만료 정책 필수
- 알림톡 템플릿 검수 시 파일 링크의 목적과 문구가 정보성 범위를 벗어나지 않게 작성

### 간단 보안안: 단일 토큰 URL + 수신자 전화번호 확인

복잡한 로그인/OTP 없이 간단히 운영하려면 다음 구조가 현실적이다.

```text
https://dl.doflab.com/t/{message_access_token}
```

알림톡 본문에는 파일 링크만 넣고, 비밀번호는 같이 보내지 않는다. 링크를 누르면 다운로드 페이지가 열리고, 페이지에서 수신자 전화번호를 입력하게 한다. 입력한 전화번호를 서버가 발송 당시 저장해 둔 수신번호와 비교한 뒤 일치할 때만 파일을 보여준다.

권장 흐름:

1. 파일 발송 시 메시지별로 긴 랜덤 토큰을 생성한다.
2. DB에는 원문 토큰이 아니라 `token_hash`, `file_id`, `message_id`, `recipient_phone_hash`, `expires_at`, `download_count`를 저장한다.
3. 알림톡에는 `https://dl.doflab.com/t/{token}` 형태의 링크만 넣는다.
4. 링크 페이지에서 전화번호 입력을 받는다.
5. 서버는 전화번호를 숫자만 남기도록 정규화한 뒤 해시 비교한다.
6. 일치하고 만료/폐기/시도횟수 제한을 통과하면 파일을 내려준다.

이 방식의 장점:

- URL을 우연히 맞추는 공격은 긴 랜덤 토큰으로 막는다.
- 링크만 전달받은 사람은 원 수신자 전화번호를 모르면 파일을 보기 어렵다.
- 같은 카카오톡에 "암호는 1234"처럼 링크와 암호를 같이 넣는 방식보다 낫다.
- 구현이 포털 로그인이나 OTP보다 단순하다.

필수 운영 조건:

- 토큰은 최소 128bit 이상, 권장 192~256bit 랜덤값을 사용한다.
- URL에 전화번호, 주문번호, 고객명, 파일명 같은 개인정보/업무정보를 넣지 않는다.
- 전화번호는 DB에 원문 저장을 피하고 해시 또는 암호화 저장을 우선 검토한다.
- 실패 응답은 항상 "잘못되었거나 만료된 링크"처럼 통일해 파일 존재 여부를 노출하지 않는다.
- 토큰당 전화번호 입력 실패 5회 정도에서 잠금 또는 지연을 건다.
- 링크는 7~30일 등 업무 성격에 맞춰 만료시킨다.
- 다운로드 접근 로그를 남긴다.
- 다운로드 페이지에는 외부 analytics/script/font를 넣지 않고, `Referrer-Policy: no-referrer`, `Cache-Control: no-store`를 적용한다.

한계:

- 수신자 전화번호를 아는 사람은 통과할 수 있다.
- 수신자가 카카오톡 링크를 직접 전달하면서 전화번호도 알려주면 막을 수 없다.
- 계약서, 민감 개인정보, 고액 결제 문서처럼 강한 본인확인이 필요하면 OTP 또는 고객 포털 로그인이 필요하다.

따라서 견적서, 거래명세서, 일반 안내자료는 `긴 랜덤 링크 + 전화번호 입력 + 만료 + 실패 제한`으로 충분히 단순하고 현실적인 균형이다. 더 민감한 문서는 별도 인증 단계로 올린다.

## 도입 전 확인 체크리스트

1. 현재 발송 타입이 알림톡(ATA)인지, 과거 친구톡(CTA/CTI) 코드인지, 브랜드 메시지인지 확인한다.
2. 과거 친구톡 코드라면 2026년 현재 브랜드 메시지 자유형으로 자동 전환되는지 발송 로그의 타입/단가를 확인한다.
3. 일반 파일 첨부 요구인지, 이미지 노출 요구인지 분리한다.
4. 일반 파일이면 파일 다운로드 링크 생성/만료/권한 정책을 먼저 설계한다.
5. 이미지면 알림톡 템플릿 이미지형 또는 브랜드 메시지 이미지형 중 메시지 성격에 맞게 선택한다.
6. 파일 링크 보안은 단일 토큰 URL + 수신자 전화번호 입력 확인을 기본안으로 검토한다.
7. 대체발송을 원하지 않으면 `kakaoOptions.disableSms: true` 설정을 검토한다.
8. 대체발송을 원하면 `from`에 사전 등록 발신번호를 넣고 LMS 비용 45원/건 가능성을 예산에 반영한다.
