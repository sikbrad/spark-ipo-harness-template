# 카카오톡 기업 상담 인입 확인 및 응대 방법 리포트

작성일: 2026-06-17  
확인 범위: 카카오 공식 비즈니스/채널/상담톡 문서와 공식 앱/고객센터 링크
최신 기준 메모: 2025-12-08 카카오 공지 기준으로 카카오비즈니스 관리자센터, 카카오톡 채널 관리자센터, 카카오맵 매장관리가 **카카오비즈니스 파트너센터**로 통합 개편되었습니다. 다만 카카오 공식 상담톡/채널 문서에는 채널 관리자센터 직접 링크로 `center-pf.kakao.com`도 계속 노출됩니다.

## 결론 먼저

카카오톡 기업 고객 상담은 설정 방식에 따라 보는 곳이 다릅니다.

1. **카카오톡 채널의 기본 1:1 채팅을 쓰는 경우**
   - PC 최신 시작점: [카카오비즈니스 파트너센터 로그인](https://business.kakao.com/login)
   - 채널 관리자 직접 진입: [카카오톡 채널 관리자센터](https://center-pf.kakao.com/)
   - 경로: 채널 선택 -> **1:1 채팅** 또는 **채팅**
   - 모바일: **카카오비즈니스 파트너센터 앱**에서 채팅 확인

2. **상담톡을 정식으로 쓰는 경우**
   - 카카오 공식 문서 기준으로, 상담톡은 API/접근 권한 상품이며 **고객센터용 상담 화면 자체는 카카오 제공 범위가 아닙니다**.
   - 상담톡을 정식 사용하면 기존 카카오톡 채널 관리자 웹/앱의 채팅 메뉴가 비활성화될 수 있습니다.
   - 따라서 인입 상담은 카카오 관리자센터가 아니라, 계약/연동된 **상담톡 공식 딜러사 또는 상담 솔루션 화면**에서 봐야 합니다.
   - 즉, Zendesk 연동으로 쓰는 중이라면 상담원은 보통 **카카오 채널 관리자센터가 아니라 Zendesk Agent Workspace**에서 상담을 봅니다.

3. **챗봇이 붙어 있는 경우**
   - 챗봇에서 **상담직원 연결** 플러그인을 누른 상담은 채널이 쓰는 상담 방식에 따라 `1:1 채팅` 또는 `상담톡`으로 넘어갑니다.

실무적으로는 먼저 [카카오비즈니스 파트너센터 로그인](https://business.kakao.com/login) 또는 [카카오톡 채널 관리자센터 직접 링크](https://center-pf.kakao.com/)로 들어가서 해당 채널의 **1:1 채팅/채팅** 메뉴가 살아 있는지 확인하세요. 메뉴가 없거나 채팅이 비활성화되어 있고 “상담톡” 계약이 있다면, 이미 연동된 외부 상담 솔루션 URL/계정이 따로 있을 가능성이 큽니다.

## 실제 Zendesk 화면 기준 판별

사용자 제공 화면 기준으로는 **카카오톡 상담이 Zendesk 티켓으로 들어오고 있는 상태**로 판단됩니다.

확인 근거:

- 티켓 제목이 `... 님과의 카카오톡 대화` 형태입니다.
- 티켓 상단에 `카카오톡을 통해`라고 표시됩니다.
- 좌측 태그에 `kakaotalk`, `matrixchat`이 붙어 있습니다.
- 우측 앱 패널에 `Matrix Chat1.1`이 표시되고, 카카오톡 말풍선 형태의 상담 내역이 보입니다.
- 대화 본문과 우측 Matrix Chat 패널의 내용이 서로 동기화되어 있습니다.

따라서 이 환경은 “카카오 채널 관리자센터에서만 보는 기본 1:1 채팅”이라기보다, **Matrix Chat 연동 앱이 카카오톡 상담을 Zendesk 티켓/대화로 생성하고 동기화하는 구조**로 보는 것이 맞습니다. 상담원은 Zendesk Agent Workspace 안에서 응대하고, Matrix Chat 앱 패널은 카카오톡 대화 원문/세션 상태를 보여주는 역할을 합니다.

이 경우 API 수집도 Zendesk 기준으로 접근하는 것이 적절합니다.

- 티켓 단위: Zendesk Tickets API
- 대화/댓글 단위: Ticket Comments API 또는 Conversation Log API
- 증분 수집: Incremental Ticket Event Export
- 카카오톡 상담 식별: `kakaotalk`, `matrixchat` 태그 또는 티켓의 `via.channel`/채널 표시값

단, Matrix Chat 앱이 별도 API를 제공한다면 첨부파일, 원문 메타데이터, 세션 종료 상태 등은 Zendesk API보다 Matrix Chat API/관리자 기능에서 더 정확하게 얻을 수 있습니다. 우선 Zendesk API로 티켓과 댓글을 가져오고, 부족한 필드가 있으면 Matrix Chat 벤더 문서/API를 확인하는 순서가 좋습니다.

### 카카오톡 화면의 `1:1 채팅` 문구 해석

고객 카카오톡 화면에 `1:1 채팅을 요청해 주셔서 감사합니다`, `상담직원과 채팅 중`처럼 표시되는 것은 **고객 입장에서 채널과 1:1 대화 세션이 열렸다는 뜻**입니다. 이 문구만으로 운영자 백엔드가 카카오 채널 관리자센터인지, Zendesk/Matrix Chat인지 결정되지는 않습니다.

이번 화면은 다음 이유로 Zendesk/Matrix Chat 연동 쪽 해석이 더 강합니다.

- 고객 화면에는 1:1 채팅 문구가 보이지만, 운영자 화면에는 Zendesk 티켓이 생성되어 있습니다.
- Zendesk 티켓 태그에 `kakaotalk`, `matrixchat`이 붙어 있습니다.
- Zendesk 우측 앱 패널이 `Matrix Chat1.1`이고, 카카오톡 말풍선 원문과 세션 종료 상태를 보여줍니다.
- 고객 화면의 자동 안내문에 “상담 품질 관리를 위해 채팅 내용은 (주)DOF에 저장됩니다”라고 표시되어 있어, 단순 카카오 관리자센터만이 아니라 회사 상담 시스템에 저장/연동되는 흐름임을 시사합니다.

따라서 이 케이스에서는 **고객 화면에는 `1:1 채팅`이라고 뜨지만, 상담원 운영 화면은 Zendesk + Matrix Chat**으로 보는 것이 맞습니다. 카카오 채널 관리자센터의 기본 1:1 채팅 메뉴에도 동시에 남는지는 별도 확인 사항이지만, 업무 처리와 API 수집의 1차 기준은 Zendesk 티켓입니다.

## 1. 기본 1:1 채팅을 쓰는 경우

### 어디서 보나

PC 최신 기준으로는 [카카오비즈니스 파트너센터 로그인](https://business.kakao.com/login)에서 진입합니다. 채널 관리자 직접 링크는 [카카오톡 채널 관리자센터](https://center-pf.kakao.com/)입니다. 로그인 후 운영할 채널을 선택하고 **1:1 채팅/채팅** 메뉴를 엽니다. 카카오 공식 채팅 가이드에 따르면 1:1 채팅 메뉴 접근 시 개인정보 보호를 위해 관리자 추가 인증이 필요할 수 있습니다.

모바일에서는 **카카오비즈니스 파트너센터 앱**을 설치해 같은 카카오계정으로 로그인하면 됩니다. 앱 소개에는 채팅 메뉴에서 카카오톡 이용자들과 실시간 대화할 수 있다고 안내되어 있습니다.

### 상담 인입 확인

채팅 상태를 켜 둔 상태에서 사용자가 메시지를 보내면 관리자 화면의 상단 채팅 알림 영역과 1:1 채팅 메뉴 우측에 읽지 않은 메시지 합계가 표시됩니다. PC 알림은 크롬 브라우저 웹 푸시 기반이므로, PC에서 놓치지 않으려면 브라우저 알림 허용이 필요합니다.

### 상담 처리 흐름

1. [카카오톡 채널 관리자센터](https://center-pf.kakao.com/) 로그인
2. 운영할 채널 선택
3. **1:1 채팅/채팅** 메뉴 진입
4. `읽지 않은 상담`, `답변하지 않은 상담`, `답변한 상담`, `완료된 상담` 필터로 인입 건 확인
5. 채팅방 클릭 후 텍스트, 이미지, 동영상, 파일 등으로 답변
6. 필요 시 자주 쓰는 답변, 쿠폰 보내기, 채널추가 요청 사용
7. 상담이 끝나면 **상담 완료하기** 처리
8. 추적이 필요한 건은 채팅방 이름, 라벨, 메모를 남김

공식 가이드 기준으로 채팅 목록은 상담 진행 상태, 라벨, 담당자, 기간 기준으로 필터링할 수 있습니다. 채팅방 관리 화면에서는 상담완료, 채팅방 이름 수정, 라벨 등록, 메모 입력이 가능합니다.

### 먼저 확인할 설정

| 설정 | 확인 경로 | 확인 이유 |
| --- | --- | --- |
| 채팅기능 ON/OFF | 채널 관리자센터 -> 1:1 채팅/채팅설정 | 꺼져 있으면 고객 채팅방에 메시지 입력창이 노출되지 않음 |
| 채팅가능 요일/시간 | 채팅설정 | 상담 운영 시간 외 안내 메시지 노출 |
| 채팅불가능 안내 메시지 | 채팅설정 | 부재중 고객에게 자동 안내 |
| 채팅 안내 메시지 | 채팅설정 | 친구가 아닌 사용자가 최초 접속할 때 1회 안내 |
| PC 알림 | 내 계정 정보/브라우저 알림 | 실시간 상담 누락 방지 |

| 채팅방 라벨/메모 | 채팅방 관리 | 재문의/이슈 분류 |

## 2. 상담톡을 쓰는 경우

### 상담톡이 무엇인지

상담톡은 고객이 `카카오톡 상담하기` 버튼, 카카오톡 채널 검색, 메시지 수신 화면 등에서 상담 요청을 하면 시작되는 **상담 메시지 API 상품**입니다. 고객은 모바일/PC 카카오톡에서 기업 채널에 말을 걸고, 상담원은 카카오 시스템과 연결된 상담원용 채팅상담화면에서 응답합니다.

다만 공식 가이드가 중요한 제한을 명시합니다. 상담톡은 API와 카카오 시스템 접근 권한 상품이고, **고객센터용 채팅 상담 시스템 자체 구축은 상담톡 서비스 제공 범위에 포함되지 않습니다.** 즉 “카카오 공식 관리자 화면 하나만 열면 모든 상담톡을 처리한다”가 아닙니다.

### 상담톡이면 어디서 보나

상담톡을 정식으로 쓰는 시점부터 기존 카카오톡 채널 관리자 웹/앱의 채팅 메뉴는 비활성화되고, 기존 채팅 이력과 이용자별 메모도 더 이상 조회할 수 없다는 것이 공식 가이드의 설명입니다.

따라서 상담톡으로 인입된 상담은 보통 아래 중 하나에서 확인합니다.

- 상담톡 공식 딜러사가 제공한 상담 솔루션 관리자 화면
- 회사가 이미 쓰고 있는 컨택센터/CRM/AICC 화면
- 채널톡, Zendesk류 외부 상담 솔루션에 상담톡이 연동된 화면
- 자체 개발 상담 시스템의 상담원 화면

확인해야 할 내부 정보는 **“우리 카카오 채널의 상담톡 공식 딜러사/연동 솔루션이 어디인가”**입니다. 이 정보는 카카오 채널 관리자센터만 봐서는 바로 안 보일 수 있으므로, 채널 세팅 담당자나 기존 홈페이지 상담 버튼을 붙인 담당자에게 확인해야 합니다.

### 브라우저로 접근 가능한가

질문한 “카카오 채널에 구독한 유저가 버튼을 눌러 상담으로 들어온 경우”는 두 가지 가능성이 있습니다.

| 고객이 누른 버튼/흐름 | 실제 사용 기능 | 상담원이 보는 곳 |
| --- | --- | --- |
| 카카오 채널 기본 채팅/문의 버튼 | 채널 기본 `1:1 채팅` | [카카오톡 채널 관리자센터](https://center-pf.kakao.com/)의 `1:1 채팅/채팅` 메뉴 |
| 챗봇의 `상담직원 연결` 버튼, 채널이 기본 1:1 채팅 사용 | 채널 기본 `1:1 채팅`으로 전환 | [카카오톡 채널 관리자센터](https://center-pf.kakao.com/)의 `1:1 채팅/채팅` 메뉴 |
| `카카오톡 상담하기` 버튼 또는 상담톡 계약 채널의 상담 인입 | `상담톡` | Zendesk, 채널톡, 공식 딜러사 솔루션, 자체 상담 시스템 등 외부 상담원 화면 |

따라서 **정식 상담톡이면 카카오 채널 관리자센터 브라우저에서 상담 대화에 접근할 수 없다고 보는 것이 맞습니다.** 카카오 공식 가이드는 상담톡 정식 사용 시 기존 카카오톡 채널 관리자 웹/앱 채팅 메뉴가 비활성화되고 기존 채팅 이력/이용자별 메모도 더 이상 조회할 수 없다고 설명합니다.

다만 “유저가 카카오 채널에서 상담 버튼을 눌렀다”는 사실만으로 `상담톡`이라고 단정하면 안 됩니다. 같은 사용자 행동처럼 보여도, 채널 설정이 기본 `1:1 채팅`이면 채널 관리자센터에서 보이고, `상담톡`/Zendesk 연동이면 Zendesk에서 보입니다.

### Zendesk와 연동된 경우 어디서 보나

Zendesk로 연동되어 있다면 운영자는 보통 Zendesk 브라우저 화면에서 상담을 봅니다.

- 상담원 화면: `https://{회사서브도메인}.zendesk.com/agent`
- 관리자 설정: `https://{회사서브도메인}.zendesk.com/admin`
- API 베이스: `https://{회사서브도메인}.zendesk.com/api/v2/...`

Zendesk 공식 문서상 KakaoTalk은 Sunshine Conversations 계열 채널로 분류되며, 티켓 via type에도 `kakaotalk`이 별도 채널로 정의되어 있습니다. 즉 Zendesk에 정상 연동된 카카오톡 상담은 Zendesk 안에서 티켓/대화로 관리되고, API로도 조회할 수 있습니다.

### Zendesk API로 가져올 수 있나

가능합니다. 이 경우 “카카오 상담톡 API를 직접 호출해서 가져온다”기보다, **Zendesk에 생성된 카카오톡 상담 티켓과 대화 이벤트를 Zendesk API로 가져오는 방식**이 현실적입니다.

대표 조회 방식은 아래와 같습니다.

```http
GET https://{subdomain}.zendesk.com/api/v2/tickets/{ticket_id}
GET https://{subdomain}.zendesk.com/api/v2/tickets/{ticket_id}/comments
GET https://{subdomain}.zendesk.com/api/v2/tickets/{ticket_id}/conversation_log
```

대량/주기 수집은 Incremental Exports를 씁니다.

```http
GET https://{subdomain}.zendesk.com/api/v2/incremental/ticket_events?start_time={unix_time}&include=comment_events
```

수집 설계는 다음이 적절합니다.

1. Zendesk에서 카카오톡 상담 티켓의 채널 식별값 확인
   - `via.channel = kakaotalk`
   - 또는 트리거/태그 예: `channel_kakaotalk`, `kakao`, `sunshine_conversations`
2. 신규/변경 티켓은 Incremental Ticket Event Export로 수집
3. 티켓별 전체 대화는 `conversation_log` 또는 `comments`로 보강
4. 첨부파일은 comment/attachment의 URL을 별도 저장
5. 상담 내용은 개인정보/분쟁 대응 정책에 맞게 보관 기간과 접근 권한을 제한

필요 권한은 Zendesk 관리자 권한 또는 API 토큰/OAuth 권한입니다. Zendesk Admin Center에서 API 토큰 접근이 꺼져 있으면 먼저 켜야 합니다.

### 상담톡 상담 흐름

공식 가이드 기준 상담톡은 아래 흐름입니다.

1. 이용자가 `카카오톡 상담하기` 버튼을 누르거나 채널에서 상담을 시작
2. 이용자의 카카오톡에서 채팅방이 열림
3. 이용자가 상담원에게 말을 걸면 상담 세션 시작
4. 상담원 화면에 신규 상담 요청 표시
5. 상담원이 응대
6. 이용자가 채팅방을 나가거나 일정 기간 채팅 수발신이 없으면 세션 종료
7. 세션 종료 후에는 이용자가 다시 먼저 말을 걸기 전까지 상담원이 먼저 말을 걸 수 없음

상담톡은 같은 이용자 식별을 위해 별도 `user_key`를 제공합니다. 다만 카카오톡 프로필명이나 프로필 이미지를 확인하는 방식은 아닙니다.

## 3. 챗봇이 붙어 있는 경우

챗봇 관리자센터는 [챗봇 관리자센터](https://chatbot.kakao.com/)입니다.

카카오 공식 챗봇 튜토리얼에 따르면, 봇에 연결된 채널이 채널 관리자센터의 1:1 채팅 또는 상담톡 기능을 사용할 경우 **상담직원 연결** 기능을 사용할 수 있습니다. 사용자가 챗봇에서 해결하지 못한 문의를 상담직원 연결로 넘기면, 해당 채널의 상담 방식에 따라 1:1 채팅 또는 상담톡 쪽으로 넘어갑니다.

실무에서는 챗봇 폴백 블록이나 주요 메뉴에 상담직원 연결 버튼이 들어가 있는지 확인해야 합니다.

6. 메뉴가 없거나 비활성화되어 있으면 상담톡 연동 여부 확인

### A-1. Zendesk 연동 여부 확인

### B. 테스트 상담 넣어보기

2. 채널 홈에서 채팅/문의/상담 버튼 클릭
3. 테스트 메시지 발송: `상담 테스트입니다. 확인 부탁드립니다.`
4. PC 관리자센터와 모바일 앱에서 인입 여부 확인
5. Zendesk Agent Workspace에서도 새 티켓/대화가 생기는지 확인
6. 채널 관리자센터에는 안 보이고 Zendesk에만 보이면 상담톡/Zendesk 연동 상담으로 판단
7. 둘 다 안 보이면 채팅 기능 OFF, 상담 가능 시간, 챗봇/상담톡 버튼 연결, Zendesk 채널 연동 상태를 확인

## 5. 운영 시 주의사항

### 일반 1:1 채팅

- 고객이 먼저 채팅을 시작해야 합니다.
- 채팅 가능 시간과 불가능 안내 메시지를 꼭 설정해야 합니다.
- 관리자 여러 명이 동시에 접속하면 같은 채팅 내용이 동일하게 업데이트됩니다.
- 1:1 대화 내용은 소비자 권익 보호와 분쟁처리를 위해 카카오톡 서버에 3년간 보관된다고 공식 가이드에 안내되어 있습니다.
- 상담 종료 후에는 반드시 `상담 완료하기`를 눌러 미처리 건과 구분하세요.

### 상담톡

- 상담톡 사용 중에는 기존 카카오톡 채널 관리자 웹/앱 채팅이 비활성화될 수 있습니다.
- 이 경우 카카오 채널 관리자센터 브라우저에서는 상담 대화를 보지 못하고, Zendesk/딜러사/자체 상담 솔루션 브라우저에서 봐야 합니다.
- 상담톡 API를 통한 1:1 채팅 수발신 내역만 상담톡에서 조회할 수 있습니다.
- 알림톡, 친구톡, 단체/그룹 메시지, 자동응답 API 챗봇 수발신 내용이 모두 상담톡 화면에 자동으로 보이는 구조는 아닙니다.
- 개인정보를 수집하는 상담이면 개인정보 수집/이용 동의 및 개인정보 처리방침/위수탁 표시가 필요합니다.
- 계약 종료 등으로 상담톡 사용을 중단하면 홈페이지 등에 노출된 `카카오톡 상담하기` 버튼도 제거해야 합니다.

### Zendesk API

- Zendesk API로 카카오톡 상담 티켓과 댓글/대화 로그를 가져올 수 있습니다.
- 단, Zendesk에 실제로 티켓이 생성되어야 하며, API 권한이 있는 계정/API 토큰이 필요합니다.
- 카카오톡 원문 전체가 항상 별도 원천 데이터처럼 보장되는 것은 아니고, Zendesk에 적재된 티켓/댓글/대화 이벤트 기준으로 가져옵니다.
- 채널 구분은 `via.channel = kakaotalk`, Zendesk 트리거의 `Ticket > Channel = KakaoTalk`, 또는 운영 중인 태그/필드 기준으로 잡는 것이 좋습니다.

## 6. 추천 운영 세팅

1. **상담 인입 장소 확정**
   - 1:1 채팅이면 `center-pf.kakao.com`
   - 상담톡/Zendesk 연동이면 `*.zendesk.com/agent` 또는 외부 상담 솔루션 URL/계정

2. **담당자 역할 구분**
   - 1차 확인 담당자
   - 답변 담당자
   - 기술/계약 이슈 에스컬레이션 담당자

3. **알림 체계**
   - PC 크롬 알림 허용
   - 모바일 앱 설치
   - 상담톡 솔루션의 이메일/앱/브라우저 알림 활성화

4. **응대 템플릿**
   - 배송/계약/결제/제품 문의별 자주 쓰는 답변 등록
   - 상담톡 솔루션을 쓰는 경우 솔루션 내부 매크로/템플릿 등록

5. **상담 종료 기준**
   - 답변 완료 후 `상담 완료`
   - 재확인 필요 건은 라벨/메모
   - 고객정보가 필요한 건은 개인정보 동의 절차 사용

## 7. Zendesk/Matrix Chat 연동에서 기본 1:1 채팅으로 바꾸려면

목표가 “Zendesk/Matrix Chat에서 응대하지 않고, 카카오톡 채널 관리자센터의 기본 `1:1 채팅`으로 직접 응대”라면 아래 항목이 바뀌어야 합니다.

### 1. 현재 연동 경로 중단

먼저 Matrix Chat/Zendesk 연동을 누가 제공하는지 확인해야 합니다. 화면상 `Matrix Chat1.1`, `matrixchat` 태그가 있으므로 Matrix Chat 앱 또는 그 벤더가 카카오톡 대화를 Zendesk 티켓으로 생성하고 있을 가능성이 큽니다.

변경할 것:

- Matrix Chat 앱의 카카오톡 채널 연동 중단
- Zendesk 내 Matrix Chat 앱 비활성화 또는 채널 연결 해제
- Zendesk 트리거/자동화에서 `kakaotalk`, `matrixchat` 태그 처리 중단 또는 보관 처리
- 신규 카카오톡 상담이 Zendesk 티켓으로 생성되지 않도록 설정 변경
- 기존 Zendesk 상담 이력은 보존/조회 정책 결정

주의:

- 현재 연동이 카카오 `상담톡` 기반이면, 카카오 공식 가이드상 상담톡 정식 사용 중에는 기존 카카오톡 채널 관리자 웹/앱의 채팅 메뉴가 비활성화될 수 있습니다.
- 따라서 단순히 카카오 채널 관리자센터에서 `1:1 채팅 ON`만 누르는 것으로는 안 될 수 있고, Matrix Chat/상담톡 계약 또는 연동을 먼저 해제해야 합니다.

### 1-1. 기본 1:1 채팅과 상담톡을 동시에 쓸 수 있는가

같은 카카오톡 채널에서 **카카오 채널 관리자센터 기본 1:1 채팅**과 **상담톡**을 동시에 운영하는 방식은 공식 문서 기준으로 어렵다고 보는 것이 맞습니다.

근거:

- 카카오 상담톡 공식 가이드는 상담톡을 정식 사용하는 시점부터 기존 카카오톡 채널 관리자 웹/앱에서 지원하는 채팅 메뉴가 비활성화된다고 설명합니다.
- 즉 상담톡 인입은 카카오 채널 관리자센터의 기본 1:1 채팅함이 아니라, 상담톡 API와 연결된 외부 상담 솔루션 또는 자체 상담 시스템에서 처리하는 구조입니다.

가능한 것과 불가능한 것을 나누면 아래와 같습니다.

| 구분 | 가능 여부 | 설명 |
| --- | --- | --- |
| 같은 채널에서 기본 1:1 채팅함과 상담톡 상담함을 동시에 열어두기 | 사실상 불가 | 상담톡 정식 사용 시 기존 채널 관리자 웹/앱 채팅 메뉴가 비활성화됨 |
| 챗봇/버튼/시나리오를 쓰고, 상담원 연결만 상담톡으로 넘기기 | 가능 | 고객은 버튼을 누르고 분기한 뒤 상담원 연결 시 Zendesk/Matrix Chat 같은 상담톡 솔루션으로 넘어갈 수 있음 |
| 상담톡을 유지하면서 채널 홈, 프로필, 단체 메시지 발송 등 다른 채널 기능 사용 | 가능 | 공식 가이드는 채팅 메뉴 외 기타 기능은 그대로 사용 가능하다고 설명함 |
| 회사가 별도 카카오 채널을 만들어 한 채널은 기본 1:1, 다른 채널은 상담톡으로 운영 | 운영상 가능성 있음 | 다만 고객 혼선, 채널 인증/검색/친구 수 분산, 개인정보 안내, 내부 운영 정책을 별도로 검토해야 함 |

따라서 “1:1도 하고 상담톡도 하고”를 원한다면 보통은 **같은 채널에서 두 상담함을 병행**하는 의미가 아니라, 아래 둘 중 하나로 설계해야 합니다.

1. **상담톡/Zendesk를 유지하고, 고객 화면의 버튼/챗봇 분기를 더 잘 정리한다.**
   - 고객은 카카오톡에서 버튼을 누릅니다.
   - 상담원 연결이 필요한 순간 Zendesk/Matrix Chat으로 들어옵니다.
   - 운영자는 Zendesk에서 계속 처리합니다.

2. **채널을 분리한다.**
   - 예: 공식 대표 채널은 상담톡/Zendesk 유지, 별도 테스트/내부/소규모 채널은 기본 1:1 채팅 운영
   - 단, 고객이 어느 채널로 문의해야 하는지 혼동하지 않도록 채널명, 버튼, 안내문, 홈페이지 링크를 분명히 분리해야 합니다.

현재 DOF 화면처럼 Zendesk 안에 Matrix Chat 앱이 붙어 있고 카카오톡 대화가 티켓으로 생성되는 구조라면, 실무적으로는 **기본 1:1 채팅을 추가로 켜기보다 상담톡/Zendesk 구조 안에서 버튼 분기와 상담 운영 규칙을 손보는 방향**이 더 안전합니다.

### 2. 카카오 채널 기본 1:1 채팅 활성화

카카오 공식 가이드 기준으로 1:1 채팅 관련 설정은 [카카오톡 채널 관리자센터 > 1:1 채팅]에서 진행합니다.

변경할 것:

- 채널 관리자센터에서 `1:1 채팅` 기능 ON
- 채팅 가능 요일/시간 설정
- 채팅 불가능 안내 메시지 설정
- 최초 진입 안내 메시지 설정
- 관리자 추가인증 및 관리자 권한 정비
- PC 알림/모바일 앱 알림 설정
- 자주 쓰는 답변, 라벨, 담당자 운영 기준 설정

운영 결과:

- 상담원은 Zendesk가 아니라 [카카오톡 채널 관리자센터](https://center-pf.kakao.com/) 또는 카카오비즈니스 파트너센터 앱에서 상담합니다.
- 고객은 계속 카카오톡 안에서 1:1 채팅처럼 보지만, 상담원 백엔드가 Zendesk에서 카카오 관리자센터로 바뀝니다.

### 3. 챗봇/메뉴/버튼 연결 변경

고객 화면에 서비스 선택 버튼이 보이므로 챗봇 또는 자동응답/시나리오가 연결되어 있을 가능성이 있습니다.

변경할 것:

- 챗봇의 `상담직원 연결` 플러그인이 기본 1:1 채팅으로 넘어가도록 확인
- Matrix Chat/Zendesk로 넘기는 별도 Webhook/API/시나리오가 있으면 제거 또는 비활성화
- 채팅방 메뉴, 간편 채팅, 서비스 선택 버튼의 연결 대상 점검
- “상담 품질 관리를 위해 채팅 내용은 (주)DOF에 저장됩니다” 같은 안내문 문구 수정 여부 검토

카카오 공식 챗봇 가이드는 채널이 채널 관리자센터의 1:1 채팅 또는 상담톡 기능을 사용할 경우 상담직원 연결이 가능하다고 설명합니다. 따라서 전환 후에는 상담직원 연결이 카카오 채널 관리자센터의 1:1 채팅으로 들어오는지 테스트해야 합니다.

### 4. 고객 입장에서 달라지는 점

기본 1:1 채팅으로 바꾼다고 해서 고객이 카카오톡에서 회사 채널에 말을 거는 것 자체가 막히는 것은 아닙니다. 채널 관리자센터에서 1:1 채팅이 ON이고 채팅 가능 상태라면 고객은 계속 카카오톡에서 메시지를 보낼 수 있습니다.

달라질 수 있는 것은 아래입니다.

| 항목 | 현재 Matrix Chat/Zendesk 연동 | 기본 1:1 채팅 전환 후 |
| --- | --- | --- |
| 카카오톡 채팅방 진입 | 계속 가능 | 계속 가능 |
| 고객이 메시지 보내기 | 가능 | 1:1 채팅 ON이면 가능 |
| 상담직원과 채팅 중 표시 | 표시될 수 있음 | 표시될 수 있음 |
| 서비스 선택 버튼 | Matrix Chat/챗봇/시나리오가 만들었을 가능성 | 동일 버튼을 유지하려면 카카오 챗봇/채팅방 메뉴/간편 채팅 쪽으로 재구성 필요 |
| 자동 안내문 | 현재 연동 시스템 문구일 가능성 | 카카오 채널 관리자센터의 채팅 안내/불가능 안내 메시지로 재작성 필요 |
| 상담 종료 처리 | Matrix Chat/Zendesk 세션 종료 로직 | 카카오 1:1 채팅의 상담완료/운영 기준으로 변경 |
| 상담 이력 확인 | Zendesk 티켓/Matrix Chat에서 관리 | 카카오 채널 관리자센터 1:1 채팅 중심으로 관리 |

즉, 고객 입장에서 가장 큰 변화는 “버튼을 누를 수 있느냐”보다 **현재처럼 서비스 선택 버튼과 자동 분류 흐름이 유지되느냐**입니다. Matrix Chat이 서비스 선택 버튼, 문의 유형 분기, 세션 종료 메시지를 만들고 있다면, 연동 해제 후에는 그 버튼/분기 UI가 사라지거나 다르게 보일 수 있습니다.

버튼을 계속 쓰려면 전환 전에 아래를 준비해야 합니다.

- 현재 버튼이 Matrix Chat 앱에서 만든 것인지, 카카오 챗봇에서 만든 것인지 확인
- Matrix Chat 버튼이면 카카오 챗봇/채팅방 메뉴/간편 채팅으로 동일 메뉴 재구성
- “구강스캐너 기술지원 / 밀링기·랩스캐너 기술지원 / 툴·소모품 주문 / 기타문의” 같은 분류를 카카오 기본 운영에서 어떻게 받을지 결정
- 전환 후 고객 카카오톡에서 버튼 표시, 버튼 클릭 후 상담 인입, 부재중 안내, 상담 종료 문구까지 실제 테스트

고객 경험을 최대한 유지하려면 “Zendesk/Matrix Chat 끄기”보다 먼저 **고객 화면에 보이는 버튼과 자동 안내를 카카오 쪽 기능으로 재현할 수 있는지** 확인해야 합니다.

### 5. 데이터/API 운영 방식 변경

Zendesk/Matrix Chat에서 기본 1:1 채팅으로 바꾸면 API와 데이터 흐름도 바뀝니다.

바뀌는 점:

- Zendesk API로 새 카카오 상담 티켓을 수집하던 방식은 더 이상 신규 상담에는 맞지 않을 수 있습니다.
- 기본 카카오 채널 1:1 채팅은 상담 내용을 Zendesk 티켓처럼 표준 API로 쉽게 가져오는 구조가 아닙니다.
- 상담 이력 검색, 통계, 담당자별 처리량, 내부 메모, SLA, 자동 분류 등 Zendesk 기능을 잃을 수 있습니다.

따라서 기업 고객 관점에서는 전환 전에 아래 결정을 해야 합니다.

- Zendesk에 상담 이력을 계속 남길 필요가 있는가?
- 카카오 관리자센터의 기본 통계/메모 기능만으로 충분한가?
- 상담 내용 백업/보관을 어떤 방식으로 할 것인가?
- 상담 품질 관리, 개인정보 보관, 분쟁 대응에 필요한 로그를 어디에 둘 것인가?

### 6. 전환 테스트 순서

1. 업무 외 시간 또는 테스트 채널에서 Matrix Chat/Zendesk 연동 중단 테스트
2. 카카오 채널 관리자센터에서 1:1 채팅 ON
3. 개인 카카오톡 계정으로 회사 채널에 테스트 메시지 발송
4. 카카오 채널 관리자센터 `1:1 채팅`에 인입되는지 확인
5. Zendesk에 새 티켓이 생성되지 않는지 확인
6. 챗봇 서비스 선택 버튼과 상담직원 연결 버튼 테스트
7. 운영 시간/부재중 메시지/알림/담당자 분배 확인
8. 기존 Zendesk 티켓 처리 중인 상담이 있다면 종료 또는 이관 후 전환

### 결론

기술적으로 바꾸려면 가능합니다. 다만 기업 고객이라면 기본 1:1 채팅 전환은 “간단해지는 대신 Zendesk 기반 운영/통계/API/이력 관리 기능을 포기하는 변경”입니다. 비용 절감이나 단순 운영이 목적이면 검토할 수 있지만, 상담량이 많거나 이력/통계/API가 중요하면 Matrix Chat/Zendesk 구조를 유지하는 편이 운영상 유리합니다.
