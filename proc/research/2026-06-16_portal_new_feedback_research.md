# 포탈 최근 NEW 피드백 2건 확인

작성일: 2026-06-16
작성 목적: 포탈 관리자 피드백 목록에 남아 있는 최근 `NEW` 피드백 2건이 무엇인지 확인한다. 개발, 데이터 수정, 상태 변경은 하지 않는다.

## 결론

최근 `NEW` 피드백 2건은 실제 업무 요청이라기보다 **피드백/알림 기능 검증 중 남은 테스트성 레코드**로 보인다.

근거는 다음과 같다.

- 작성자가 둘 다 `관리자(테스트)` / `admin-test--@doflab.com`이다.
- 피드백 유형은 둘 다 `SUGGESTION`이지만, 메시지는 각각 `asdf`, `asdfasdf`뿐이다.
- 발생 페이지가 둘 다 `/admin/feedback`이다. 실제 주문/수금/계약 화면에서 올라온 업무 피드백이 아니다.
- 더 최신 1건에는 `재발송 버튼 검증용 관리자 메모입니다.`라는 관리자 메모와 `notificationSentAt` 값이 있다.
- 같은 날 생성된 피드백은 이 2건뿐이다.

## 확인 대상

읽기 전용으로 포탈 운영 DB의 `Feedback` 테이블을 조회했다. 관리자 피드백 목록 구현 기준상 기본 정렬은 `createdAt desc`이고, `status = 'NEW'` 필터를 적용했다.

## 주변 정황

`Notification` 테이블에는 2026-06-15 11:52:27 KST에 `FEEDBACK` 알림 1건이 생성되어 있다.

- 제목: `피드백 알림`
- 메시지: `재발송 버튼 검증용 관리자 메모입니다.`
- 링크: `/admin/feedback`
- 수신 사용자: 위 피드백 작성자와 같은 테스트 관리자 계정

다만 `Notification` 테이블은 피드백 ID를 직접 들고 있지 않고 링크/메시지/사용자 기준으로만 남는다. 따라서 최신 피드백 1번과의 연결은 시각, 링크, 수신자, 메모 내용이 일치한다는 점에서의 추정이다.

## 추가 확인

- 2026-06-15 00:00 UTC 이후 생성된 `Feedback` 레코드는 위 2건뿐이었다.
- 현재 `NEW` 상태 피드백은 총 3건이 조회되었고, 위 2건을 제외한 1건은 2026-06-04 KST의 수금 화면 관련 피드백이다.
- 이번 작업에서는 포탈 코드, 포탈 DB, 피드백 상태를 변경하지 않았다.

## 판단

운영상 긴급히 분석해야 할 사용자 업무 피드백은 아니다. 필요하면 별도 요청으로 `DISMISSED` 또는 `DONE` 처리할 수 있지만, 이번 요청 범위가 "리서치문서만"이므로 상태 변경은 하지 않았다.
