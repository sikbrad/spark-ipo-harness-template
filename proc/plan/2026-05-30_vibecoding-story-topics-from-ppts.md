# 바이브코딩 이야기 꼭지 발굴

## 목표

지정된 Google Drive 백업 경로와 OneDrive Issues 경로의 바이브코딩 관련 PPT를 다시 찾아 활성 슬라이드 중심으로 읽고, 사용자가 가진 이야기 꼭지를 가능한 많이 발굴해 출처와 함께 Markdown 문서로 정리한다. 이후 로컬 Notion의 바이브코딩/강연 자료를 추가해 같은 문서에 살을 붙인다.

## 작업 원칙

- 새 API 스크랩은 하지 않는다.
- 파일 발견과 슬라이드 텍스트 추출에는 도구를 쓰되, 꼭지 판단과 병합은 직접 읽고 수행한다.
- Google Drive File Provider의 대용량 PPT 복사는 피하고, 필요하면 symlink 또는 원본 직접 읽기를 우선한다.
- 개인 정보, 회사 내부 정보, 수강생 정보, 계좌, 비공개 링크는 공개용 본문에 넣지 않고 리스크로 표시한다.
- 말라카 외부 강의, 회사 AX/사내교육, 개인 서사/퓨처셀프를 구분한다.
