# Threads 예약 삭제와 재작성 발행

## 목표
기존 예약 글을 모두 삭제하고, 이전 예시 문체를 그대로 복제하지 않도록 로컬 바이브코딩/AX 자료를 바탕으로 새 6~7월 Threads 글을 재작성한다. 오늘 글은 즉시 발행하고, 나머지는 07:00~09:00 사이 랜덤 시각으로 예약한다.

## 작업 항목
- [x] Threads 예약 글 목록/삭제 경로 확인
- [x] 기존 예약 글 삭제
- [x] 로컬 자료와 기존 게시글/초안 중복 축 재검토
- [ ] 새 글감/문체 원칙 작성
- [ ] 오늘 즉시 발행 글 작성 및 게시
- [ ] 2026-06-11~2026-07-31 예약 초안 작성
- [ ] 초안 3블록/500자/민감정보 검증
- [ ] 07:00~09:00 랜덤 시각 예약
- [ ] 예약/삭제 결과 보고

## 재작성 원칙
- 사용자가 준 예시는 리듬 참고만 한다. 같은 문장 구조, 같은 소재 배열, 같은 질문 패턴을 반복하지 않는다.
- 기존 6월/7월 초안에서 이미 쓴 주제 표현과 직접 중복되지 않게 한다.
- 개인 실명, 회사명, 고객명, 내부 프로젝트명, 내부 URL, 조직 상황은 쓰지 않는다.
- `AX 리더`, `바이브코더`, `실무 자동화 담당자`의 추상적 고민으로 쓴다.

## 삭제 결과
- 예약 목록 URL: `https://www.threads.com/drafts`
- 삭제 대상: 기존 2026-06-10~2026-06-30 예약 글 21건
- 결과: `No drafts yet` 화면 확인

## 새 문체 방향
- 예시의 줄바꿈과 3블록 체인 구조만 참고한다.
- 첫 문장 패턴을 매번 `요즘...`으로 시작하지 않는다.
- 소재를 `AI가 잘한다/위험하다` 일반론보다 `작업 운영`, `검토 회의`, `작은 실험`, `자료 손질`, `조직 확산`의 다른 장면으로 분산한다.
- 질문형 마무리는 일부에만 사용하고, 나머지는 판단 문장이나 체크 기준으로 끝낸다.
