# Teams 인물 업무규칙 보강

## 목표
새로 만든 주문/제품/국가/제품군 처리 매트릭스를 김규탁, 조소연, 정재회 업무규칙 문서에 반영한다. 기존 규칙의 판단 축은 유지하되, 실제 주문 사례에서 드러난 보류 조건과 담당 경계를 더 촘촘히 만든다.

## 대상
- `data/teams/peoples/업무규칙/README.md`
- `data/teams/peoples/김규탁/업무규칙/`
- `data/teams/peoples/조소연/업무규칙/`
- `data/teams/peoples/정재회/업무규칙/`

## 보강 원칙
- 원형 데이터와 전수장부는 수정하지 않는다.
- 매트릭스의 주문번호는 규칙의 근거로만 사용하고, 규칙 문서는 확인값/보류조건/다음 행동 중심으로 쓴다.
- 김규탁은 출고 실행 가능성, 조소연은 주문/ERP/계약 정합성, 정재회는 창고/채널 실행 상태 중심을 유지한다.
- 개인식별 정보와 계정성 값은 규칙 문서에 재기재하지 않는다.

## 검증 메모
- 대상 업무규칙 문서와 계획 문서가 모두 비어 있지 않음을 확인했다.
- `주문-제품-국가-제품군`, `재고 배분`, `Freedom HD`, `Premill`, `DOF USA`, `PIM KIT`, `HASS`, `온라인 취소`, `포장/라벨`, `대체 규격`, `생산중단`, `택배 마감`, `패킹 완료`가 보강 대상 규칙 문서에서 검색되는 것을 확인했다.
- 개인 연락 수단, 회사 메일 형식, 계정성/임직원성/대화 식별/생년 정보 등 민감 식별 패턴이 새 규칙 문서와 계획 문서에 없음을 확인했다.
- `git diff --check` 결과 공백/패치 오류가 없었다.
- `data/`는 `.gitignore` 대상이므로 Git diff에는 데이터 문서 변경이 표시되지 않을 수 있다.

## 추가 보강 메모
- 사용자 추가 요청: 가격, 배송, 인코텀즈 등에서 실제로 어떤 값이 달라졌고 어떤 정정 지시가 나왔는지 업무규칙에 남길 것.
- 처리 방식: 인물별 `업무규칙/값차이-정정지시-규칙.md`를 추가하고, README와 업무유형 규칙에서 연결한다.
- 김규탁: 품목코드 `KR-D10-01`, 운송장 `3PKG` vs `5PKG/50KG`, HS `9031` vs `9021.29`, DAP/EXW/DHL 운임 변경, 송금통화/주문통화, CIPL/포장수량 차이를 정정 규칙으로 추가했다.
- 조소연: 단가/수량 반전, 계약금액 `2,200만원` vs `1,100만원`, 부가세 포함 `110만원` vs `121만원`, 금액 0 해외 주문의 인코텀즈 필수, DAP/EXW/DDP 오류, Buyer/VAT 서류값, 전표 재입력 규칙을 추가했다.
- 정재회: 주문수량/현재고 차이, `16mm` vs `20mm` 대체 규격, 주문 `758개` vs 패킹 `755개`, 온라인 입금/취소, 배송지 오기입, 회수 완료 수량, 택배 마감 차이를 추가했다.
- 추가 검증: 세 `값차이-정정지시-규칙.md` 파일이 비어 있지 않고, 요청 키워드 검색이 확인됐으며, 민감정보 패턴 없음과 `git diff --check` 통과를 확인했다.
