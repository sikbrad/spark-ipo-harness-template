# 포탈 그룹 멘션 대상 확장 계획

## 목표

포탈 채터와 할일 댓글에서 `물류`, `회계` 같은 권한 그룹을 @멘션 대상으로 선택할 수 있게 한다.

완료 기준:
- 채터 멘션 후보에서 활성 사용자와 권한 그룹이 함께 검색된다.
- 그룹 멘션은 화면에 `@물류`, `@회계`처럼 보이고, 저장 마크업은 사용자 멘션과 구분된다.
- 서버는 그룹 멘션을 해당 그룹의 활성 사용자들로 풀어 알림과 이메일을 발송한다.
- 기존 사용자 멘션과 과거 댓글 렌더링은 깨지지 않는다.

## 현상 확인

- 할일 담당자 선택은 이미 그룹을 지원한다.
  - `todos.getAssigneeOptions`는 활성 사용자와 `CapabilityGroup` 전체를 함께 내려준다.
  - `TodoAssigneePicker`는 그룹 항목을 `GROUP:{groupUk}` 값으로 만들고, 검색어에 그룹명/role/userCount를 포함한다.
  - 할일 서버는 담당자가 그룹이면 `UserGroup`을 조회해 실제 담당 사용자 목록으로 확장한다.
- 채터/할일 댓글 멘션은 현재 사용자 전용이다.
  - `MentionDropdown`은 `usersApi.getUsers`만 호출하고 후보 타입이 `{ uk, name }` 사용자로 고정되어 있다.
  - `ChatterPanel`은 선택한 멘션을 `@[이름](user:{userUk})`로 저장한다.
  - 댓글 서버는 정규식 `@\[(...)\]\(user:(...)\)`만 파싱해서 `mentionedUserUks` 배열을 만든다.
  - 댓글 알림 리스너도 `mentionedUserUks`를 그대로 `Notification.userUk`로 사용한다.
- 로컬 DB 기준으로 그룹 자체는 존재한다.
  - `물류 / LOGISTICS`: 8명
  - `회계 / ACCOUNTING`: 4명
  - 사용자 검색에서 `물류`로 걸리는 것은 테스트 사용자 `물류담당(테스트)` 1명뿐이라, 현재 UI에서 보이는 것은 그룹이 아니라 사용자다.

## 판단

가능하다. 다만 프론트 후보 목록에 그룹을 단순히 추가하는 패치로는 부족하다.

현재 댓글 멘션 저장/파싱/알림 계약이 모두 `user` 전용이므로, 그룹 멘션은 별도 타입으로 저장하고 서버에서 그룹 소속 활성 사용자로 확장해야 한다. 이렇게 해야 `@물류`라는 표시와 실제 알림 대상이 일치하고, 그룹 uk를 사용자 uk처럼 잘못 저장하는 위험을 피할 수 있다.

권장 저장 마크업:
- 사용자: `@[홍길동](user:{userUk})` 유지
- 그룹: `@[물류](group:{groupUk})` 신규

## 구현 시 주의점

- 기존 댓글의 `@[이름](user:uk)` 마크업은 그대로 렌더링되어야 한다.
- 그룹명을 바꿔도 과거 댓글 표시가 유지되도록 저장 시점의 표시명을 댓글 본문에 남기는 현재 방식을 유지한다.
- 알림 수신자는 그룹 자체가 아니라 그룹의 활성 사용자다.
- 그룹 후보는 현재 모든 활성 사용자 보유 `CapabilityGroup`을 대상으로 연다. `물류`, `회계`만 특례 처리하지 않는다.

## 검증 기록

- 첫 Playwright E2E 실행은 서버 미기동으로 `/api/v1/auth/me` 프록시가 `ECONNREFUSED`가 되어 실패했다.
- 로컬 서버 기동 후 E2E를 재실행했고, 할일 댓글 `@물류` 그룹 멘션 저장/렌더링/알림 검증까지 통과했다.
- 추가로 `playwright-cli` 브라우저에서 `/companies/1` 채터 입력창에 `@물류`를 입력해 그룹 후보가 사용자 후보와 함께 표시되는 것을 확인했고, 선택 후 댓글 저장 시 `@물류`가 그룹 멘션 스타일로 렌더링되는 것을 확인했다.
