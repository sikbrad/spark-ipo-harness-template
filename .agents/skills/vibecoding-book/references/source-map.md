# Vibecoding Book Source Map

Use this reference when deciding where to pull local evidence from.

## Primary Local Sources

- `data/vibecoding-book/extracts/`: extracted text from latest PPT/PPTX decks. Start with `all-slides.json`, `decks.json`, and `slide-inventory.json`.
- `data/vibecoding-book/research/`: Notion-derived author profile and topic evidence from prior runs.
- `data/vibecoding-book/proposal/`, `final/`, `rewrite-*`, `connected-rewrite-*`, `market-rewrite-*`: previous publication plans, manuscripts, render reviews, and known bad directions to avoid.
- `data/daily/*/summary.md`: daily narrative summaries.
- `data/daily/*/raw/notion-jot.json`: DailyJot raw notes.
- `data/daily/*/raw/voice-*.txt`: transcribed recordings; useful for voice, career, lecture, and AX field scenes.
- `data/notion/md/**/*.md`: rendered Notion markdown, especially `Note DB`.
- `data/notion/hydrated/pages/*.json`: hydrated Notion pages with page blocks.
- `data/db/chatgpt.sqlite`: conversation title index. Query titles before opening raw exports.
- `output/teams/conversations/**/*.md` and `data/daily/*/raw/teams-*.json`: company/AX collaboration context. Use only when relevant and summarize cautiously.

## Separation Rules

- Malaka lecture material: external lectures, cohort design, KUBIT/POC clinic, student success screens, setup/login issues, result sharing.
- Company AX material: internal education, DOF/CRM/ERP/order/ledger work, Teams/Outlook coordination, company clinics, KPI, ownership, verification.
- Author biography: career and motivation. Use it to explain credibility, not to dominate the first chapters.
- Production workflow: keep HTML/CSS/PDF/book-making details in reports unless the user explicitly asks for a book about publishing workflow.
